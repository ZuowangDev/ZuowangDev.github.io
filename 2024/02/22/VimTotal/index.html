Vim:the Best Editer

可以用来编辑任何文本文件，未保存的文件会保留在缓存区。
现代计算机中，vi命令链接着?im。
文件名最好由字母和_组成
~表示没有行，甚至没有建立空行。
三个模式：

  1. 命令模式。
  2. 插入模式。
  3. ex 命令模式。（不常用）

退出保存方式

:wq
ZZ

擦除本次打开文件的所有编写

:e!

擦除所有编写并退出

:q!

!表示强制执行

:w表示将改变后的存入新文件，:w!表示覆盖原文件
:w后加上新文件的地址

缓冲区已满，使用ex命令:pre(:preserve的简写)，用其他软件删除

以下是Vim基本操作。

· 移动光标。
· 简单编辑操作。
· 加入新行。
· 更多进入插入模式的方式。
· 模式转换。

进入文档时，默认进入命令模式，按i进入文档模式，ESC即进入命令模式。

光标移动命令：

h 左一空格
j 下一行
k 上一行
l 又一空格
e/E:光标移动到单词结尾。
(+/ctrl+P:上一行；-/ctrl+N:下一行)
在不同的电脑上，箭头发送的信息不同，因此选择字母。
行数与回车数相关，即使某一行在屏幕上占据了很多行。
w:后退一小格，W:后退一大格（以空格为分界）
b:向前一小格，B:向前一大格（以空格为分界）
数字+G:移动到第“数字”行，G:移动到文末
a:add,c:change;d:delete a character;x:delete;r:replace;y:yank;p:put;o:open a new line
cw:删除至本单词结尾；c2b:删除前面两个单词；c$:删除至逻辑行末；c0:删除至逻辑行首。

General Form
such as c2b,(command)(number)(text object)
cc:replace all of th line
键入发生在c命令影响不至整行时，空白行发生在影响至少一行时。
例如，在使用cw之后，会进入a状态，因为在曾经的vi中，需要使用插入命令才能刷新出c命令。
C = c$
替换命令：r+character
单个字符版的c命令：s命令：对字符进行操作，之后进入插入模式。
R:进入持续替换模式，直到输入ESC。
~进行大小写字母之间的转换
dw:删除至下一个单词前的空格（包括）
w/W:包不包括标点
dd:删除正行，不进入插入模式
D:删除至行末，不进入插入模式
u:撤回上一条命令
但是，您仍然可以使用p命令恢复最近的删除，因为vi将最后九个删除保存在九个编号的删除寄存器中。例如，如果您知道第三次删除是您想要恢复的，请键入：33p
可以通过删除某行之后再粘贴到另一个位置完成文本的移动。
您可以使用xp（删除字符并放在光标后面）来转置两个字母。例如，在单词mvoe中，字母vo被转置（反转）。要纠正换位，请将光标放在v上，然后按x，然后按p。巧合的是，transpose这个词可以帮助你记住序列xp；x代表trans，p代表pose
yy复制本行
.:周期命令，重复上一个命令

更多插入文本方式：
A：在行末附加；I：在行头插入；o：在光标所在下一行打开新行；
O：在光标所在位置上一行打开新行；s：删除光标所在位置的字符后再替换文本；
S：删除一整行后再替换文本；R：用新的字符覆盖原有字符。
输入50iESC会插入50个星号，输入25a-ESC则会插入50个字符(25对星号与连字符的组合)
合并两行：J+.
命令复习：在chapter2的章末。

chapter 3 快速移动位置

  • 根据屏幕来移动。
  • 根据文本块移动。
  • 根据搜索模式的结果来移动。
  • 根据行号来移动。

根据屏幕移动

：向下滚动一整屏幕；：向上滚动一屏幕；
^D：向下滚动半屏；^U：向上滚动半屏。
z enter：光标移动到屏幕顶端并滚动屏幕；
z.：光标移动到屏幕中间，滚屏；
z-：光标移动到屏幕底端，滚屏。
滚屏会重画屏幕；ctrl + L会只重画不滚动。
H：移动到屏幕顶端的行；
M：移动到屏幕中间的行；
L：移动到屏幕底端的行；
nH/nL：移动到顶端往下/底端往上的第n行。

行内移动。

  • ^ 移动到当前行第一个非空格处。
  • n| 移动到当前行的第n列。

根据文本块移动。

\ 开头 结尾
单词 e E
句子 ( )
段落 { }
节 [[ ]]

vim搜索模式。

搜索命令：/
往回搜索：?
继续搜素操作：n：同向；N：反向。

vim同样支持行内搜索：

\ 向前 向后
本处 fx Fx
相差一个c tx Tx

重复上一个搜索命令：
;：同向。 ,： 反向。

根据行号来移动。

xG：光标转至x行第一位置。

输人两个反引号(``)会回到原来的位置(即上一次使用G命令时所在的位置)，除非
你在中间做了编辑操作。如果你做了编辑操作，然后用G以外的命令移动了光标，*
则: 会将光标移回到你上一次做编辑操作时的地方。如果你使用了搜索命令(/或?)，则
会带你回到上一次使用搜索命令的地方。一对引号(‘’)与两个反引号的功用一样，不 同之处在于，它把光标移到前次位置所在行的开头，而不是确实的位置上。

命令扩展。

更多命令组合。

P53,表格。
命令格式：(command)(number)(text object)
打开文件后直接跳转到某个单词的位置：vim +/“” text-name
只读模式：vim -R file
恢复缓冲区：vim -r file
最近删除的会进入缓冲区1号，共有9个缓冲区

26个文本复制缓冲区
"dyy：将当前行复制到d缓冲区中。
"a7yy：将后续7行复制到缓冲区a中。
"zy)：将光标处到句子末的文本删除并放入z缓冲区中。

标签设置。

针对vim文本的编辑，可以将某处设为标签,在别处编辑后，可以通过命令回到标签处
mx：将当前位置标记为x
返回书签处：'x：移动到书签所在行的第一个字符；`x：将光标移动到以x为标记的字符；‘’：回到上一个标记所在行的第一个字符；``：回到上一个标记的确切位置

vim + file：在最后一行打开文件
vim -c command file：打开后执行命令，通常是行号或者是搜索命令
详细见P60
