Compound Types

reference and point

reference

  1. 引用一旦创建必须进行初始化，无法重新绑定到其他object。
  2. A reference is not an object. Instead, a refernce is just another name for an already existing object.

Pointers

A compound type that “points to” another type.
A pointer need not be initialized at the time it is defined.

指针指向的四种类型:

  1. Object
  2. 指向对象的末尾
  3. 空指针
  4. 无效指针

指针一定要进行初始化，否则会导致程序崩溃。可以先定义指针指向的值，或者指向null或0；

指针可以使用 == 和 != 符号。

void * 指针可以指向任意object，但是无法通过该指针访问和操作指向的object，必须进行强制类型转换才可以。

*只绑定紧跟它的一个变量名。

Const Qualifier

Can not be changed. By default, const objects are local to a file. The compile will replace uses of the variable with its corresponding value during compilation.

当某个const需要别的文件也可以调用时，可以在.h文件中进行声明，在cc文件中进行定义，增加extern限定。

** reference and const **

对与const的引用，同样不能通过修改引用的值。const refernce is a refernce to const

引用必须绑定到一个固定的地址，但是使用const修饰的引用会给右侧创建一个临时变量，再将这个引用变量指向这个临时变量。非常量引用不能绑定到临时变量。一个const引用指向的可能是一个非const的值，const只是限定了通过引用可以进行的操作。

** pointers and const **

同样，指向const值的指针无法用于修改其值。如果一个指针定义为指向常量的指针，但所指向的变量不是常量，只是无法通过指针修改它，但仍能通过其他方式修改指向的值。

const pointers

指针本身就是常量，指针指向的地址一旦确定就无法更改。

C++中复杂变量的声明可以从右往左读理解，即从变量名开始阅读

Top-Level Const

Top-Level Const: 指针本身是常量；Low-Level Const: 指针指向常量。

** constexpr **

  1. 值不可变
  2. 编译期可计算

常见的常量表达式：1. 字面量 2. 使用常量表达式初始化的const对象

在C++等语言中，部分语法场景会强制要求使用常量表达式，如数组初始化。C++11标准中，增加了constexpr标识符，通过对变量进行声明，可以要求编译器验证该变量是否是常量表达式。

在constexpr中可是使用的类型有限，被称为“字面值类型”。包括算术类型、引用类型和指针类型，IO类型和string类型都不是字面值类型。

Dealing with Types

所需的数据类型会变复杂：1. 数据类型定义复杂；2. 数据类型难以确定。

一些解决类型定义复杂度的方法

Type Alias

使用typedef进行类型的别名自定义

Auto

自动类型分配。auto依赖于初始化器的分配，因此必须给auto变量进行初始化。进行多变量声明时，多个变量必须为同一基础类型，如int，float。

** auto + const **: 如果auto指定的变量初始化时，底层是const, const属性被保留；顶层是const，则const属性被忽略。

Decltype

希望变量的类型由某个表达式推导得到，又不想用这个表达式初始化这个变量。使用decltype作为类型说明符，能返回其操作数的类型。编译器仅分析其表达式类型，不进行实际求值。

与const的区别：decltype保留顶层const的数据类型，保留完整的数据类型。

decltype判断表达式返回类型的依据（普通类型和引用类型）：通俗判断依据是，表达式的结果能否放在赋值运算符的左侧？如果能被赋值，则decltype会返回该对象类型的引用。
